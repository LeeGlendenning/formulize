Data handling for text element
  • {NAME} will print full name
  • {UNAME} will print user name
  • {EMAIL} will print user email
  • {ID} will cause the entry ID number of the entry to be inserted into the textbox, when the entry is first saved.
  • {SEQUENCE} will cause the values in the box to be a series of consecutive numbers.
  • PHP Code (ending with the line '$default = $something;') will be interpreted to generate the default value. PHP Code is the only situation where more than one line of this box will be read. In your PHP code, you can use $form_id to get the ID number of the form, and $entry_id to get the ID number of the particular entry that the user is editing.

If you select another form element here, then text that users type into this element will be compared with values entered in the other element. If a match is found then the text users type into this element will be clickable in the "List of Entries" screen, and will take users to the matching entry in the other form.

Unique value

Users must enter a unique value into this box (no duplicates allowed).

Required field

Users will be required to enter a value for this field.

Database storage of element

Elements that will only contain numbers should use a numeric type, so that sorting and calculations work optimally.

This is an advanced option that you can use to control the MySQL datatype that is used in the underlying database field for this element. The value in ( ) shows which datatype will be used.

If you don't know what all this means, then just accept the defaults. Formulize can intelligently select appropriate values for regular text boxes based on the 'numbers only' setting, and the number formatting options.
